Interface Data Types

Within the load-balancing algorithms, objects are identified by a global identification (ID) value provided by the application. This global ID must be unique across all processors. The application may also provide a local ID value that it can use for faster location of objects within its own data structure. For example, pointers to objects' data may be provided as the local IDs; the pointers can then be used to directly access data in the query functions. The load balancing library does not use these local IDs, but since it must pass them to the application in the interface query functions, it must store them with the objects' data. The data types of the global and local IDs (LB_GID and LB_LID, respectively) are application-defined through an include file, lb/lb_user_const.h. In the source code distribution, the default data types are int for LB_GID and void* for LB_LID; the application developer can change these types before compiling the library.

Since LB_GID and LB_LID may be defined as arrays or structures, macros that allow assignment and comparison of global and local IDs must also be defined in lb/lb_user_const.h. These macros should be used in all load-balancing algorithm development; no assumption about data types for IDs should be made. The macros provided in lb/lb_user_const.h are listed in the table below. The LB_SET_GID and LB_SET_LID macros perform assignment of global and local IDs, respectively. The remaining macros perform comparisons of global IDs; since local IDs are not used in the load-balancing algorithms, comparison macros are not needed for them.
 

Definition Description
LB_GID The data type of the unique global IDs used by the application.
LB_LID The data type of the local IDs used by the application query functions.
LB_SET_GID( a, b) Assigns the global ID value in b to a (i.e., a = b).
LB_SET_LID( a, b) Assigns the local ID value in b to a (i.e., a = b).
LB_EQ_GID( a, b) Returns TRUE if global IDs a and b are equal; returns FALSE otherwise.
LB_NE_GID( a, b) Returns TRUE if global IDs a and b are not equal; returns FALSE otherwise.
LB_LT_GID( a, b) Returns TRUE if global ID a is less than global ID b; returns FALSE otherwise.
LB_LE_GID( a, b) Returns TRUE if global ID a is less than or equal to global ID b; returns FALSE otherwise.
LB_GT_GID( a, b) Returns TRUE if global ID a is greater than global ID b; returns FALSE otherwise.
LB_GE_GID( a, b) Returns TRUE if global ID a is greater than or equal to global ID b; returns FALSE otherwise.
Data type and macro definitions that must be provided to Zoltan by the application in lb/lb_user_const.h.
