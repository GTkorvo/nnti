Object List Function

Most algorithms in Zoltan obtain information about all objects located on a processor. To provide this information, an application can register either an ZOLTAN_OBJ_LIST_FN query function or an ZOLTAN_FIRST_OBJ_FN / ZOLTAN_NEXT_OBJ_FN query function pair. Algorithms can call these query functions to obtain object information.

However, because this operation is common to many algorithms, Zoltan provides a single function that can be called from an algorithm to obtain a list of object IDs and weights (if appropriate). This function, Zoltan_Get_Obj_List, provides a uniform method of calling the registered query functions supplied by the application. Its use simplifies new algorithm development and code maintenance in Zoltan. An example of its use can be found in rcb/shared.c.

void Zoltan_Get_Obj_List( struct Zoltan_Struct *zz, ZOLTAN_ID_PTR global_ids, ZOLTAN_ID_PTR local_ids, int wdim, float *objwgts, int *ierr);
Zoltan_Get_Obj_List returns arrays of global and local IDs (and object weights, if OBJ_WEIGHT_DIM is not zero) for all objects on a processor. It is a convenient function that frees algorithm developers from calling an ZOLTAN_OBJ_LIST_FN query function or an ZOLTAN_FIRST_OBJ_FN/ ZOLTAN_NEXT_OBJ_FN query function pair directly.
 
Arguments:
    zz A pointer to the Zoltan structure created by Zoltan_Create.
    global_ids Upon return, an array of global IDs of objects on the current processor.
    local_ids Upon return, an array of local IDs of objects on the current processor. NULL is returned when NUM_LID_ENTRIES is zero.
    wdim The number of weights associated with an object (typically 1), or 0 if weights are not requested.
    objwgts Upon return, an array of object weights. Weights for object i are stored in objwgts[(i-1)*wdim:i*wdim-1]. If wdim is zero, the return value of objwgts is undefined and may be NULL.
    ierr Error code.
Required Query Functions:
ZOLTAN_NUM_OBJ_FN
ZOLTAN_OBJ_LIST_FN or ZOLTAN_FIRST_OBJ_FN/ZOLTAN_NEXT_OBJ_FN pair
