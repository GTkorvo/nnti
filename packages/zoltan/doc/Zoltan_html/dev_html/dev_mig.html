Migration-Help Tools

The migration-help tools in the Zoltan library perform communication necessary for data migration in the application. The routine LB_Help_Migrate calls application-registered packing routines to gather data to be sent to other processors. It sends the data using the irregular communication package. It then calls application-registered unpacking routines for each imported object to add received data to the processor's data structures. See the Zoltan User's Guide for more details on the use of and interface to the migration-help tools.

Implementation of the migration-help tools is completed. However, changes to the irregular communication package may necessitate changes in the migration-help tools. For example, the irregular communication package currently requires that the number of bytes sent is uniform for each object. Thus, for each call to LB_Help_Migrate, the size of each object's data is necessarily uniform. If an application has data of varying sizes for objects, it must report the maximum data size in the LB_OBJ_SIZE_FN query function. When the irregular communication package is modified to handle non-uniform object sizes, the LB_Help_Migrate function and the LB_OBJ_SIZE_FN query function will have to be changed to take advantage of non-uniform data sizes for data migration.
