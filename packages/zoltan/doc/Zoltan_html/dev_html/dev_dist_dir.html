Layout of Directories

The source code is organized into several subdirectories within the DLB main directory. Load-balancing interface and utility routines are stored in a single directory. Communication and memory allocation utilities available to all algorithms are in separate directories. Each load-balancing method (or family of methods) should be stored in its own directory. In the following table, the source-code directories currently in the DLB directory are listed and described.
 
Directory Description
lb Interface definitions (lbi_const.h), user data type definitions (lb_user_const.h), load-balancing data structure definitions (lb_const.h, lb.h), interface functions (lb.c), and functions related to the load-balancing interface (lb_util.c, lb_util_const.h). See Interface Functions, Interface Data Types, and Data Structures.
all Memory allocation functions (all_allo.c, all_allo_const.h). Utility functions common (or possibly common) to several algorithms (all_util.c, all_util_const.h).
comm Bruce Hendrickson and Steve Plimpton's unstructured communication package (comm.c, comm_const.h). 
par Parallel computing utilities (par_const.h, par_util.c).
param Routines for changing parameter values at runtime.
rcb Steve Plimpton's implementation of Recursive Coordinate Bisection (RCB).
oct RPI's octree partitioning and space-filling curve algorithms. See Octree Partitioning and Space Filling Curves.
gr Some graph manipulation tools using tree-based and hash-table based implementations of graphs. These tools are only partially developed and are not compiled into the library.
vx Some vertex manipulation routines that work with the graph routines in the gr directory. These routines are only partially developed and are not compiled into the library.
id Some object ID manipulation routines that work with the graph routines in the gr directory. These routines are only partially developed and are not compiled into the library.
ch Routines developed by Bruce Hendrickson and Rob Leland to read Chaco input files and build graphs using the graph routines in the gr directory. These routines are only partially developed and are not compiled into the library.
driver Test driver program, zdrive.
docs/Zoltan_html Zoltan documentation and home page.
docs/Zoltan_html/ug_html User's guide in HTML format.
docs/Zoltan_html/dev_html Developer's guide in HTML format.
The directory structure of the Zoltan distribution.
