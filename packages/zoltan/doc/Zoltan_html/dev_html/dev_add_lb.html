Load-Balancing Function Implementation

The new load-balancing algorithm should be implemented as an LB_FN. The type definition for an LB_FN is in lb/lb_const.h and is described below. When the new algorithm is selected in LB_Set_Method, the LB_Fn field of the LB_Struct is set to point to the LB_FN function for the new algorithm. This pointer is then used in invoking load balancing in LB_Balance.
 

typedef int LB_FN(struct LB_Struct *lb, int *num_import, LB_GID **import_global_ids, LB_LID **import_local_ids, int **import_procs, int *num_export, LB_GID **export_global_ids, LB_LID **export_local_ids, int **export_procs); 

The LB_FN function type describes the arguments passed to a load-balancing function. The input to the function is an LB_Struct containing pointers to application-registered functions to be used in the load-balancing algorithm. The remaining arguments are output parameters listing the objects to be imported or exported to the processor in the new decomposition. The arrays for global and local IDs and source processors must be allocated by the load-balancing function. The load-balancing function may return either the import arrays, the export arrays, or both. If no import data is returned, *num_import must be set to a negative number, and similarly with *num_export.
 
Parameters:
    lb A pointer to the LB_Struct to be used in the load-balancing algorithm.
    num_import Upon return, the number of objects to be imported to the processor for the new decomposition. A negative number indicates that no import data has been computed and the import arrays should be ignored.
    import_global_ids Upon return, an array of size num_import containing global IDs of the objects to be imported to the processor for the new decomposition.
    import_local_ids Upon return, an array of size num_import containing local IDs of the objects to be imported to the processor for the new decomposition.
    import_procs Upon return, an array of size num_import containing the processor IDs of processors owning (in the old decomposition) the objects to be imported for the new decomposition. 
    num_export Upon return, the number of objects to be exported from the processor for the new decomposition. A negative number indicates that no export data has been computed and the export arrays should be ignored.
    export_global_ids Upon return, an array of size num_export containing global IDs of the objects to be exported from the processor for the new decomposition.
    export_local_ids Upon return, an array of size num_export containing local IDs of the objects to be exported from the processor for the new decomposition.
    export_procs Upon return, an array of size num_export containing  the processor IDs of processors owning (in the old decomposition) the objects to be exported for the new decomposition. 
Returned Value:
    int Error code.
