Memory Management in Zoltan Load-Balancing Algorithms

Zoltan contains a set of memory management routines. It is strongly recommended that load-balancing algorithm developers use LB_Malloc and LB_Freeinstead of the standard C routines for most memory management.

When a dynamic structure needs to be returned to the application, however, special memory allocation routines are needed.  For example, the import and export lists of data to migrate are returned to an application from LB_Balance. There are two special routines for managing memory for such situations, called LB_Special_Malloc and LB_Special_Free. Algorithms must use these functions to maintain compatibility with both C and Fortran90 applications; these special routines manage memory in a way that is compatible with both languages.

Some load-balancing algorithms may contain persistent data structures, that is, data structures that are preserved between calls to the load-balancing routine. The lb structure contains a field lb->Data_Structure for this purpose, allowing multiple load-balancing structures to preserve their own decomposition data. The developer should write a function that frees this data structure.  Use LB_RCB_Free_Structure as an example.

int LB_Special_Malloc(struct LB_Struct *lb, void **array, int size, LB_SPECIAL_MALLOC_TYPEtype); 

The LB_Special_Malloc routine allocates memory for result arrays like import_global_ids, import_local_ids, import_procs, export_global_ids, export_local_ids and export_procs. Since these arrays are returned to the application, they must be allocated by LB_Special_Malloc to insure they are allocated by the same language as the application. Memory allocated by LB_Special_Malloc must be deallocated by LB_Special_Free.
 
Parameters:
    lb The load balancing structure currently in use.
    array Usually of type int**, LB_GID**, or LB_LID**. Returned as a pointer to the allocated space.
    size The number of elements (not bytes) to be allocated.
    type The type of array to allocate. Must be one of LB_SPECIAL_MALLOC_INT, LB_SPECIAL_MALLOC_GID or LB_SPECIAL_MALLOC_LID.
Returned Value:
   int 1 if the allocation succeeded; 0 if it failed.
Example:
ierr = LB_Special_Malloc(lb, (void **)import_gid, num_import, LB_SPECIAL_MALLOC_GID);
Allocates an array with num_import LB_GIDs.

int LB_Special_Free(struct LB_Struct *lb, void **array, LB_SPECIAL_MALLOC_TYPE type); 

LB_Special_Free frees memory allocated by LB_Special_Malloc.
 
Parameters:
    lb The load balancing structure currently in use.
    array The array to be freed.
    type The type of the array. Must be one of LB_SPECIAL_MALLOC_INT, LB_SPECIAL_MALLOC_GID or LB_SPECIAL_MALLOC_LID.
Returned Value:
   int 1 if the deallocation succeeded; 0 if it failed.
Example:
ierr = LB_Special_Free(lb, (void **)import_gid, LB_SPECIAL_MALLOC_GID);
Frees the LB_GID array import_gid.
