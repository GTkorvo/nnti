Plan for Future Development

Support heterogeneous computing systems.
  • Three levels of support:
    • Heterogeneous processors: Vector of processor speeds; partition to match it.
    • Heterogeneous memory sizes: Vector of memory sizes; partition to not exceed it.
    • Heterogeneous networks: Graph of network; map well to it.
  • Augment existing algorithms to handle heterogeneity.
  • Insist that new algorithms handle it, too.
    • Develop a good model of heterogeneity before implementing too many algorithms.
Add Partitioning Algorithms to the Library.
  • Inertial algorithms.
    • Add heterogeneity and prefix-sum to limit memory problems.
  • Unbalanced Recursive Bisection (URB).
    • Hard to fit into current thinking about heterogeneity.
    • Requires application to be partitioned before machine.
  • Terminal propagation.
  • Empirical comparisons.
Tool for Determining New Communication Pattern.
  • Closely related to metric of partition quality.
  • Dependent upon the application.
Miscellaneous Performance Improvement Tools.
  • Mechanisms for incrementally modifying graph passed to balancer. (Consider impact on data structures before doing too many algorithms.)
  • Automated algorithm selection and go/no-go decisions.
  • Allow variable-sized objects in irregular communication package.
