Interface Functions

The interface to the dynamic load-balancing library is defined in the file lb/lbi_const.h. This file should be included in application programs that use the load-balancing library. It is also included in lb/lb_const.h, which should be included by all load-balancing library files to provide access to the load-balancing data structures described above.

In lb/lbi_const.h, the enumerated type LB_FN_TYPE defines the application query function types (e.g., LB_NUM_OBJ_FN_TYPE and LB_OBJ_LIST_FN_TYPE). The interface query routines (e.g., LB_NUM_OBJ_FN and LB_OBJ_LIST_FN) and their argument lists are defined as C type definitions (typedef). These type definitions are used by the application developer to implement the query functions needed for the application to use the load-balancing library.

Prototypes for the load-balancing and migration interface functions (e.g., LB_Balance and LB_Help_Migrate) are also included in lb/lbi_const.h. Interface functions are called by the application to register functions, select a load-balancing method, invoke load balancing and migrate data.

For more detailed information on Zoltan's query and interface functions, please see the Zoltan User's Guide.
