Migration-Help Functions

If requested, Zoltan can automatically transfer an application's data between processors to realize the new decomposition. This functionality will be performed as part of the call to LB_Balance if the AUTO_MIGRATE parameter to TRUE (nonzero) via a call to LB_Set_Param. Alternatively, the application can leave AUTO_MIGRATE FALSE and call LB_Help_Migrate itself. In either case, routines to pack and unpack object data must be provided by the application. See Migration Example for an example using auto-migration. The functions are the migration-help interface functions. Their detailed descriptions can be found below.

<type-lid> can be either INTEGER(LB_INT) or TYPE(LB_LID)  
LB_Compute_Destinations computes the inverse communication map needed to export data. That is, given, for each processor, a list of non-local objects assigned to the processor, LB_Compute_Destinations computes the list of objects each processor needs to export to other processors to satisfy their assignments of non-local objects.
 
Parameters:
    lb The load-balancing object, created by LB_Create_Object, to be used in this invocation of the migration-help routine.
    num_import  The number of objects that are needed by this processor that are stored on other processors in the old decomposition (i.e., the number of objects to be imported to this processor).
    import_global_ids An array of size num_import listing the global IDs of objects needed by this processor that are stored on other processors in the old decomposition.
    import_local_ids An array of size num_import listing the local IDs of objects needed by this processor that are stored on other processors in the old decomposition.
    import_procs An array of size num_import listing the processor IDs of the processors that own objects needed by this processor (i.e., the source processors).
    num_export Upon return, the number of objects that were stored on this processor in the previous decomposition that are needed by other processors (i.e., the number of objects that must be sent from this processor to other processors).
    export_global_ids Upon return, an array of size num_export listing the global IDs of objects to be sent from this processor.
    export_local_ids  Upon return, an array of size num_export listing the local IDs of objects to be sent from this processor.
    export_procs Upon return, an array of size num_export listing the processor IDs of processors that need the sent objects (i.e., the destination processors).
Returned Value:
    int Error code.

FORTRAN: FUNCTION LB_Help_Migrate(lb, num_import, import_global_ids, import_local_ids, import_procs, num_export, export_global_ids, export_local_ids, export_procs) 
INTEGER(LB_INT) :: LB_Help_Migrate 
TYPE(LB_Struct),INTENT(IN) :: lb 
INTEGER(LB_INT), INTENT(IN) :: num_import, num_export 
<type-gid>, POINTER, DIMENSION(:) :: import_global_ids, export_global_ids 
<type-lid>, POINTER, DIMENSION(:) :: import_local_ids, export_local_ids 
INTEGER(LB_INT), POINTER, DIMENSION(:) :: import_procs, export_procs 

<type-gid> can be either INTEGER(LB_INT) or TYPE(LB_GID) 
<type-lid> can be either INTEGER(LB_INT) or TYPE(LB_LID) 
LB_Help_Migrate takes lists of objects to be sent to other processors, along with the destinations of those objects, and performs the operations necessary to send the data associated with those objects to their destinations. If a migration pre-processing routine (LB_PRE_MIGRATE_FN_TYPE) is specified, LB_Help_Migrate first calls that function. It then calls an application-registered function (LB_OBJ_SIZE_FN_TYPE) to obtain the size of a migrating object's data. For each object, it calls an application-registered packing routine (LB_PACK_OBJ_FN_TYPE) to load communication buffers. It communicates the buffers to the destination processors. Finally, for each object received, it calls an application-registered unpacking routine (LB_UNPACK_OBJ_FN_TYPE) to move the data from the received communication buffer into the new processor's data structures.
 
Parameters:
    lb The load-balancing object, created by LB_Create_Object, to be used in this invocation of the migration-help routine.
    num_import  The number of objects that are needed by this processor that are stored on other processors in the old decomposition (i.e., the number of objects to be imported to this processor).
    import_global_ids An array of size num_import listing the global IDs of objects needed by this processor that are stored on other processors in the old decomposition.
    import_local_ids An array of size num_import listing the local IDs of objects needed by this processor that are stored on other processors in the old decomposition.
    import_procs An array of size num_import listing the processor IDs of the processors that own objects needed by this processor (i.e., the source processors).
    num_export The number of objects that were stored on this processor in the previous decomposition that are needed by other processors (i.e., the number of objects that must be sent from this processor to other processors).
    export_global_ids An array of size num_export listing the global IDs of objects to be sent from this processor.
    export_local_ids An array of size num_export listing the local IDs of objects to be sent from this processor.
    export_procs An array of size num_export listing the processor IDs of processors that need the sent objects (i.e., the destination processors).
Returned Value:
    int Error code.
