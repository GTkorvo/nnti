Load-Balancing Functions

The following functions are the load-balancing interface functions in the Zoltan library; their descriptions are included below.
LB_Set_Method
LB_Set_Param
LB_Balance
LB_Eval
LB_Free_Data

C: int LB_Set_Method(struct LB_Struct * lb, char * string); 
 
FORTRAN: FUNCTION LB_Set_Method(lb, string) 
INTEGER(LB_INT) :: LB_Set_Method 
TYPE(LB_Struct), INTENT(IN) :: lb 
CHARACTER(LEN=*), INTENT(IN) :: string 
LB_Set_Method is used to specify which load-balancing technique should be used by the application.
 
Parameters:
    lb Pointer to the load-balancing structure created by LB_Create.
    string A string containing the name of the desired load-balancing technique. Valid values of string are 
"RCB" (for recursive-coordinate bisection), 
"PARMETIS" (for any of the methods in the ParMETIS library), 
"OCTPART" (for octree partitioning), 
"REFTREE" (for refinement tree based partitioning), and 
"NONE" (for no load-balancing).
Returned Value:
    int Error code.
Default:
If LB_Set_Method is not called, RCB is used.
 
C: int LB_Set_Param(struct LB_Struct * lb, char * param_name, char * new_val); 
 
FORTRAN: FUNCTION LB_Set_Param(lb, param_name, new_val) 
INTEGER(LB_INT) :: LB_Set_Param 
TYPE(LB_Struct), INTENT(IN) :: lb 
CHARACTER(LEN=*), INTENT(IN) :: param_name, new_value 
LB_Set_Param is used to alter the value of one of the parameters used by Zoltan.  All Zoltan parameter have reasonable default values, but this routine allows a user to provide alternative values if desired.
 
Parameters:
    lb Pointer to the load-balancing structure created by LB_Create.
   param_name A string containing the name of the parameter to be altered.  Note that the string is case-insensitive.  Also, different load-balancing structures can have different parameter values. 
 
   new_val A string containing the new value for the parameter.  Possible stings include "3.154", "True", "7" or anything appropriate for the parameter being set. As above, the string is case-insensitive. 
 
Returned Value:
    int Error code.
C: int LB_Balance(struct LB_Struct * lb, int * changes, int * num_import, LB_GID ** import_global_ids, LB_LID ** import_local_ids, int ** import_procs, int * num_export, LB_GID ** export_global_ids, LB_LID ** export_local_ids, int ** export_procs); 
 
FORTRAN: FUNCTION LB_Balance(lb, changes, num_import, import_global_ids, import_local_ids, import_procs, num_export, export_global_ids, export_local_ids, export_procs) 
INTEGER(LB_INT) :: LB_Balance 
TYPE(LB_Struct), INTENT(IN) :: lb 
LOGICAL, INTENT(OUT) :: changes 
INTEGER(LB_INT), INTENT(OUT) :: num_import, num_export 
<type-gid>, POINTER, DIMENSION(:) :: import_global_ids, export_global_ids 
<type-lid>, POINTER, DIMENSION(:) :: import_local_ids, export_local_ids 
INTEGER(LB_INT), POINTER, DIMENSION(:) :: import_procs, export_procs 

<type-gid> can be either INTEGER(LB_INT) or TYPE(LB_GID) 
<type-lid> can be either INTEGER(LB_INT) or TYPE(LB_LID) 
LB_Balance invokes the load-balancing routine specified by LB_Set_Method.
 
Parameters:
    lb Pointer to the load-balancing structure, created by LB_Create, to be used in this invocation of the load-balancing routine.
    changes Set to 1 or .TRUE. if the decomposition was changed by the load-balancing method; 0 or .FALSE. otherwise.
    num_import  Upon return, the number of objects that are now assigned to this processor that were assigned to other processors in the old decomposition (i.e., the number of objects to be imported to this processor).
    import_global_ids Upon return, an array of size num_import listing the global IDs of objects to be imported to this processor.
    import_local_ids Upon return, an array of size num_import listing the local IDs of objects to be imported to this processor.
    import_procs Upon return, an array of size num_import listing the processor IDs of the processors that owned the imported objects in the previous decomposition (i.e., the source processors).
    num_export Upon return, the number of objects that were assigned to this processor in the previous decomposition that are now assigned to other processors (i.e., the number of objects that must be exported from this processor to other processors).
    export_global_ids Upon return, an array of size num_export listing the global IDs of objects to be exported from this processor.
    export_local_ids Upon return, an array of size num_export listing the local IDs of objects to be exported from this processor.
    export_procs Upon return, an array of size num_export listing the processor IDs of processors that will own the exported objects in the new decomposition (i.e., the destination processors).
Returned Value:
    int Error code.
 

C: void LB_Eval(struct LB_struct *lb, int print_stats, int *nobj, float *obj_wgt, int *cut_wgt, int *nboundary, int *nadj, int *ierr); 
 
FORTRAN: SUBROUTINE LB_Eval(lb, print_stats,  nobj, obj_wgt, cut_wgt, nboundary, nadj, ierr) 
TYPE(LB_Struct), INTENT(IN) :: lb 
LOGICAL, INTENT(IN) :: print_stats 
INTEGER(LB_INT), INTENT(OUT), OPTIONAL :: nobj, cut_wgt, nboundary, nadj, ierr 
REAL(LB_FLOAT), INTENT(OUT), DIMENSION(vwgt_dim), OPTIONAL :: obj_wgt 
LB_Eval evaluates the quality of the load balance in the load-balance structure pointed to by lb. Some quality metrics are only available if the graph query functions have been registered. LB_Eval may either print a summary of the results to stdout or return the results in the output parameters.
 
Parameters:
    lb Pointer to the load-balancing structure.
    print_stats If print_stats>0 (.TRUE. in Fortran), print a summary (maximum and sum) of the quality metrics to stdout.
    nobj Upon return, the number of objects on this processor.
    obj_wgt Upon return, an array containing the sum of object weights on this processor.
    cut_wgt Upon return, the cut size (weight) for this processor.
    nboundary Upon return,  the number of boundary objects on this processor.
    nadj Upon return,  the number of adjacent processors as defined by the communication graph.
    ierr Upon return, an error code.
Returned Value:
  None. 
 
Query functions: 
     Required:		  
LB_NUM_OBJ_FN 
LB_OBJ_LIST_FN or LB_FIRST_OBJ_FN/LB_NEXT_OBJ_FN
     Optional: LB_NUM_EDGES_FN 
LB_EDGE_LIST_FN
An output parameter is returned only if the input value of that parameter was not NULL. (An exception is that the error code is always returned.) The rationale for this feature is that if one wishes just to print the evaluation results, one can simply set all (or some of) the output parameters to NULL in the function call. From Fortran, this is achieved by omitting one or more of the optional output parameters.

C: int LB_Free_Data(LB_GID ** import_global_ids, LB_LID ** import_local_ids, int ** import_procs, LB_GID ** export_global_ids, LB_LID ** export_local_ids, int ** export_procs); 
 
FORTRAN: FUNCTION LB_Free_Data(import_global_ids, import_local_ids, import_procs, export_global_ids, export_local_ids, export_procs) 
INTEGER(LB_INT) :: LB_Free_Data 
<type-gid>, POINTER, DIMENSION(:) :: import_global_ids, export_global_ids 
<type-lid>, POINTER, DIMENSION(:) :: import_local_ids, export_local_ids 
INTEGER(LB_INT), POINTER, DIMENSION(:) :: import_procs, export_procs 

<type-gid> can be either INTEGER(LB_INT) or TYPE(LB_GID) 
<type-lid> can be either INTEGER(LB_INT) or TYPE(LB_LID) 
LB_Free_Data frees the memory allocated by the load-balancing routines to return the results of LB_Balance. Memory pointed to by the arguments is freed and the arguments are set to NULL in C or nullified in Fortran. Note that this does not destroy the load balancing data structure itself. That requires a call to LB_Destroy.
 
Parameters:
    import_global_ids The array containing the global IDs of objects imported to this processor.
    import_local_ids The array containing the local IDs of objects imported to this processor.
    import_procs The array containing the processor IDs of the processors that owned the imported objects in the previous decomposition (i.e., the source processors).
    export_global_ids  The array containing the global IDs of objects exported from this processor.
    export_local_ids The array containing the local IDs of objects exported from this processor.
    export_procs The array containing the processor IDs of processors that own the exported objects in the new decomposition (i.e., the destination processors).
Returned Value:
    int Error code. 
 
C: void LB_Destroy (struct LB_Struct ** lb); 
 
FORTRAN: SUBROUTINE LB_Destroy(lb) 
TYPE(LB_Struct), POINTER :: lb 
LB_Destroy frees the memory associated with a load-balancing structure and sets the structure to NULL in C or nullifies the structure in Fortran. Note that LB_Destroy does not deallocate the import and export arrays returned from LB_Balance. This requires a separate call to LB_Free_Data.
 
Parameters:
    lb A pointer to the address of the load-balancing structure, created by LB_Create, to be destroyed.
