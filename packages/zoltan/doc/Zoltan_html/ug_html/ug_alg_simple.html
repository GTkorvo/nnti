Simple

Simple is, as the name suggests, a very simple algorithm. It is mainly intended for testing. It uses neither geometry nor connectivity (graph/hypergraph), so it requires very few query functions. The simple strategy is as follows: Consider all objects (on all processors) as a sequence. Assign the first chunk of n/num_parts objects to the first partition, the next chunk to the second, and so on. Simple is advanced enough to generalize this method to handle vertex weights and target partition sizes.
 
Method String: Simple
Parameters:
Required Query Functions:

ZOLTAN_NUM_OBJ_FN
