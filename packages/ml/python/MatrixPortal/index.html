Welcome to the Matrix Portal, a Trilinos-based environment to test and experiment with (distributed) sparse linear algebra algorithms. The Matrix Portal allows you to generate or upload linear systems, then analyze and solve them using direct and sparse methods.

An overview of the project is as follows: you define a set of linear problems and a set of methods to solve them, then you compare the solution phases, in order to evaluate (using a criterion you specified) the effectiveness of the solution methods.

The Matrix Portal operates as a sequence of steps. Currently, there are four steps:

Step 1:
Data
 is the definition of the linear system matrices. After step 1, you have one or more "ProblemID" or labels, which identify the linear problem you want to analyze or solve.
Step 2:
Parameters
 defines the parameters for the linear algebra object analysis, direct and iterative solutions. In this step, for example, you can choose the Kyrlov accelerator and the preconditioning parameters. If you are interested in comparing the solutions, You also have to specify an evaluation criterion (for example, the number of iterations).
Step 3:
Compute
 performs all the computatons and reports the result on the web browser;
Step 4:
Results
 collects the results and offers tools to analyze and compare them.
